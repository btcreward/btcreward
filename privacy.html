App Permissions Explained

The Application requests the following device permissions, each for a specific purpose:

  • INTERNET — Required for API calls, Firebase, AdMob ads, and backend sync.
  • POST_NOTIFICATIONS — Used to send mining status & BTC reward alerts (Android 13+).
  • VIBRATE — Provides vibration feedback when rewards are earned.
  • WAKE_LOCK — Keeps the CPU active during active BTC reward simulation gameplay sessions.
  • RECEIVE_BOOT_COMPLETED — Restarts scheduled reward reminder notifications after device reboot.
  • FOREGROUND_SERVICE (dataSync) — Shows live BTC reward computation progress as a persistent notification during gameplay.
  • ACCESS_NETWORK_STATE — Checks internet connectivity before API calls and ad loading.
  • AD_ID — Required by Google AdMob for advertising on Android 13+ per Google policy.

No location, camera, microphone, contacts, or SMS permissions are used or requested.
